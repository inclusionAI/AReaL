# Branch Processing for "Alternatively" Keywords

This tool processes a JSONL training dataset to add parallel branches at
"Alternatively" keywords in assistant responses.

## How it works

1. For each assistant response with **n** occurrences of `"Alternatively"`:
   - The first **n-1** `"Alternatively"` blocks are wrapped in
     `<Parallel><Trial>...</Trial></Parallel>` tags.
   - New alternative branches are generated by feeding the conversation prefix
     (user message + assistant content before the keyword) to a **sglang** server.
   - The model generates until it finishes or produces another `"Alternatively"`.
   - The **last** `"Alternatively"` block is kept as-is (no branching).

2. The result looks like:
   ```
   [Content before Alt 1]
   <Parallel>
     <Trial>Alternatively [original block 1]</Trial>
     <Trial>[generated branch 1]</Trial>
     <Trial>[generated branch 2]</Trial>
   </Parallel>
   <Parallel>
     <Trial>Alternatively [original block 2]</Trial>
     <Trial>[generated branch 1]</Trial>
   </Parallel>
   Alternatively [last block, unchanged]
   ```

## Requirements

- Python 3.8+
- `requests` library (`pip install requests`)
- A running **sglang** server at `http://localhost:30000` (or custom URL)

## Usage

```bash
python process_branches.py \
    --input ../train.jsonl \
    --output ../train_branched.jsonl \
    --num_branches 2 \
    --sglang_url http://localhost:30000 \
    --max_tokens 2048 \
    --temperature 0.7
```

### Arguments

| Argument | Default | Description |
|---|---|---|
| `--input` | `../train.jsonl` | Input JSONL file |
| `--output` | `../train_branched.jsonl` | Output JSONL file |
| `--sglang_url` | `http://localhost:30000` | sglang server URL |
| `--num_branches` | `2` | Branches per Alternatively block (**hyperparameter**) |
| `--max_tokens` | `2048` | Max tokens to generate per branch |
| `--temperature` | `0.7` | Sampling temperature |
